## How I Build

- **User interface first:** I care about spacing, hierarchy, responsive behavior, and UI states that make interactions understandable.
- **Accessibility as part of the component:** I practice ARIA attributes, focus states, dialog behavior, mobile menus, and keyboard-friendly flows.
- **Typed React patterns:** I use TypeScript, reusable hooks, component variants, and shared utilities to keep projects easier to maintain.
- **Practical async UI:** I build with API requests, validation, loading skeletons, retry/error states, and query/state refresh patterns as I learn them.
- **Consistent tooling:** Most projects include linting, formatting, build scripts, and deployable Vercel demos.
